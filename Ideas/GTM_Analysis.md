# Portable Health Agent — GTM 分析

## 一、GTM 角度的硬問題

### 1. 誰是第一批付錢/留存的人？

別賭「全民健康管理」，太散。三個可行楔子，挑一個：

- **慢性病患家屬**（高齡父母陪診者）：痛點明確 — 記不住醫師講什麼、跨科室資訊斷裂。願付費、會主動推給家人。
- **醫療旅遊/海外就醫**（台灣人在東南亞、外籍移工）：跨國跨語言病歷攜帶，剛需，無對手。
- **罕病/多科室病患**：自己就在做 Excel 整理病歷，產品直接取代他們的土法煉鋼。

**首推 #1 慢性病家屬**：市場最大、付費意願最高、有口碑傳播路徑。

### 2. 醫師端不要做 App

醫師永遠不會裝你的 App。改成：病患掃 QR → 開瀏覽器 → 醫師看 web viewer，零安裝。

**MVP 階段更激進**：連 QR 都不要醫師掃，直接讓病患手機螢幕給醫師看，或一鍵生成 PDF 摘要 print/AirDrop。先讓病患單邊就有用。

### 3. 區塊鏈是後端裝飾，不是賣點

對 Web2 初階使用者，「上鏈」毫無說服力，反而讓人覺得「這是不是詐騙幣？」。

對外行銷話術換成：
- 「**防竄改備份**」
- 「**醫療級存證**」

Tx Hash 那一頁砍掉，或藏進「進階 → 驗證資訊」。

## 二、低摩擦前端設計建議

### 砍掉的東西

| 原 PRD | 問題 | 改法 |
|---|---|---|
| 「鏈上驗證狀態」icon 在清單 | 看不懂 | 移除，只在詳細頁底部「這份紀錄已防竄改 ✓」一行字 |
| Tx Hash 顯示 | 沒人看 | 摺疊在「進階資訊」 |
| 「事前授權」大字提示 | 嚇到使用者 | 改一句「按下後會錄音，僅你看得到」+ 小灰字法律說明 |
| 「密鑰加密」「Secure Storage」 | 術語 | 完全不出現在 UI |
| 6 個畫面 + 4 Tabs | MVP 過重 | 砍到 3 Tab：今日 / 病歷 / 我 |

### 主畫面只留一顆按鈕

```
[ 🎤  我要記錄這次看診 ]   ← 80% 螢幕、超大
   下方：上次看診摘要卡片（一張）
```

不要 Insight、不要待辦、不要儀表板。Web2 小白看到 dashboard 就退出。

### 錄音流程零選擇

- 按下 → 直接錄（權限第一次彈一次就好）
- 不要選科別、不要選醫院、不要打標籤 → AI 摘要時自己猜，猜錯讓使用者改
- 結束 → 自動處理 → 30 秒內出摘要（這段要有動畫，否則使用者會以為當機）

### 摘要確認頁只做兩件事

1. 看一眼
2. 一鍵存

不要逼使用者編輯欄位。要改？長按該段才出現編輯。預設 happy path 是「看完 → 存」兩 tap 結束。

### 分享頁的關鍵

- 預設選項：「**給這次看診的醫師看**」（最常見）
- QR Code 自動產，不要先讓使用者選範圍。範圍選擇藏在「進階」
- 加一個「另存 PDF」當逃生口 — 因為很多醫師根本不會掃 QR

### 註冊登入

- 第一次開啟不要逼註冊。Apple ID / Google 一鍵，或乾脆先讓他用，存第一筆病歷時才綁帳號（延遲註冊）
- Wallet / 助記詞？**完全不要出現**。用 passkey 或 social login 代管私鑰（MPC / TSS），失敗也讓 email 救援

## 三、上市節奏（90 天）

| 週次 | 範圍 |
|---|---|
| Week 1-4 | 只做 iOS、只做錄音→摘要→PDF匯出。**不上鏈、不分享、不醫師端**。先驗證「病患願意錄、AI 摘要夠準」。 |
| Week 5-8 | 加雲端備份（用 Web3 存證，但 UI 叫「防竄改備份」）+ QR 分享給醫師（web viewer）。 |
| Week 9-12 | 加家屬代管（一個帳號管父母病歷）→ 這是病毒擴散的關鍵功能。 |

**冷啟動通路**：慢性病社群（FB 糖尿病/高血壓社團）、長照中心、藥局合作。不要砸醫院 BD，太慢。

## 四、技術上需要對應調整的地方

- **On-device LLM 3B-8B** 在低階 Android 會卡爆。MVP 建議做「敏感資料本地遮蔽 → 雲端 LLM 摘要 → 結果存本地」的折衷。純 on-device 是 v2。
- **Solana 用什麼角色**：只存 hash + 授權 log，**不要試圖把病歷本身放鏈上**（成本、隱私都不行）。賣點寫成「使用 Solana 提供醫療級存證」對 Hackathon 評審有用，對使用者完全藏起來。
- **錄音存留**：明確規則 — 「摘要完成後 24 小時刪原始音檔」，寫進 onboarding 一頁告知。
