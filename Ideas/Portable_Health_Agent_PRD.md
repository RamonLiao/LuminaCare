# Portable Health Agent - 商業規格書 (PRD)

## 專案概述：病患主控的 AI 醫療助理
本專案旨在打造一個「以手機為中心」的 AI 醫療 agent 應用，讓病患真正拿回醫療數據主控權（Portable 病歷）。透過在地端（On-device）運行的 AI 處理敏感數據，結合區塊鏈技術確保資料不可竄改性，我們解決了隱私疑慮，並在不取代醫師判斷的前提下，協助醫療人員做出更準確的決策。

### 核心價值主張
*   **病患充權（Empowerment）**：病歷存在病患手機，醫院需獲授權才能讀取。
*   **隱私與信任**：在地端進行語音轉文字與 AI 摘要，敏感資料不上雲端。
*   **可攜式病歷（Portable Records）**：跨院、跨國就醫無縫銜接。
*   **網路韌性**：離線可用，適合偏鄉或網路不穩定的場景。

## 一、螢幕列表 (Screen List)
核心 UX 架構以「病患當下任務」為焦點，分為 4 個主 Tab，對應 6 個關鍵畫面：

1.  **Home：今日總覽** (今日狀態、重點提醒)
2.  **看診錄音中畫面** (事前授權、錄音介面)
3.  **AI 看診摘要確認畫面** (病患確認 AI 整理的重點)
4.  **「我的病歷」時間軸列表** (歷史就醫紀錄)
5.  **單一 Portable 病歷詳細頁** (包含摘要、附件、鏈上驗證狀態)
6.  **分享與權限管理頁** (QR Code 分享、歷史授權撤銷)

## 二、線框簡述 (Wireframe Brief)

### 1. Home：今日總覽
*   **下一步建議**：顯示近期回診或待辦事項（例：「查看上次看診摘要」）。
*   **快速開始**：主要 CTA 按鈕「這次看診我要記錄」。
*   **健康 Insight**：基於長期數據的 AI 洞察（例：「過去 3 週頭痛頻率增加 40%」）。

### 2. 看診錄音畫面
*   **準備錄音**：大字提示「請先徵得醫師同意」。說明錄音僅供摘要，短時間內自動刪除。
*   **錄音中**：顯示計時器與「正在為你記錄」，並具備紅色「結束錄音」按鈕。

### 3. AI 摘要確認畫面
*   **內容區塊**：主訴、醫師提問、初步評估、檢查結果、用藥指示。
*   **互動**：病患可自由補充備註。
*   **CTA**：「儲存這次看診」（觸發本地加密與上鏈）。

### 4. 「我的病歷」清單與詳細頁
*   **清單**：時間軸呈現，標示日期、科別、AI 摘要前兩行、鏈上驗證狀態。
*   **詳細頁**：完整 AI 摘要、原始檢查報告（照片/附件），並展示區塊鏈驗證資訊（Tx Hash）。
*   **CTA**：「分享這份病歷給醫師」。

### 5. 分享與權限管理頁
*   **立即分享**：選擇分享範圍（單次/特定時期/特定疾病），產生具時效性的 QR Code。
*   **歷史授權**：列出曾讀取病歷的醫師與時間，並提供「撤銷」按鈕。

### 6. AI 助理 (Tab)
*   **核心功能**：基於病患在地端病歷回答問題（例：「這個藥會想睡嗎？」）。
*   **邊界**：僅提供解釋與提醒，不作醫療診斷。

## 三、資料流架構 (Data Flow)

採用「手機端處理 + 鏈上驗證 + 後端輔助」的混合架構 (Hybrid Architecture)：

### Flow A：看診後生成與上鏈
1.  **錄音與處理**：手機 App 錄音 ➡️ 手機端 (或 Local Gateway) 執行 ASR 語音轉文字。
2.  **AI 摘要**：透過在地端 LLM (如 3B-8B 輕量模型) 產生結構化摘要。
3.  **加密與儲存**：使用病患密鑰加密摘要與附件 ➡️ 存入手機 Secure Storage。
4.  **上鏈驗證**：計算資料 Hash ➡️ 透過 OpenClaw skill 將 Hash、時間戳、醫院 ID 寫入醫療聯盟鏈。
5.  **UI 更新**：回傳 Tx Hash，標示為「已上鏈 ✅」。

### Flow B：醫師授權與讀取
1.  **病患授權**：病患選擇分享範圍，手機生成一次性 Token 或加密 QR Code。
2.  **醫師讀取**：醫師掃描 QR Code 或發送存取請求。
3.  **驗證與解密**：系統驗證 Token 有效性與鏈上授權紀錄。
4.  **展示**：手機端短暫解密對應病歷，透過 Web Viewer 渲染給醫師檢視，不允許下載明文。
